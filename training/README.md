# Agentic Engineering Training

This training path helps a new employee build from zero familiarity with the playbook to safe, high-throughput agentic engineering in a production repository with multiple developers.

The training uses a simple fictitious repo, **Acme Orders**, so learners can practice real workflows without production risk.

## Training Goals

By the end of the full training, a learner should be able to:

- explain the difference between vibe coding and agentic engineering
- turn vague work into agent-ready issues
- set up the local tools and GitHub access needed for the workflow
- use GitHub Projects to track backlog, ownership, status, risk, validation, and release state
- run a plan-first Claude Code workflow
- use branches and worktrees to isolate work
- keep PRs small and reviewable
- coordinate with other humans to avoid collisions
- apply safety and permission rules before high-risk edits
- record validation evidence accurately
- identify rollout, rollback, and production-risk concerns
- work an issue or backlog item end to end through release, verification, and cleanup
- operate as a high-throughput contributor without outrunning review capacity

## Suggested Format

Run this as a 6-day onboarding track or a 2-week part-time ramp.

Recommended cadence:

1. Read the relevant playbook section.
2. Practice the concept in Acme Orders.
3. Produce a small artifact: setup checklist, project update, issue, plan, branch, PR summary, review, validation note, release note, or rollout checklist.
4. Get human feedback.
5. Repeat with slightly more ambiguity or collaboration pressure.

The learner should not move into production-repo agentic work until they can pass the rubric in [Assessment rubric](rubric.md).
